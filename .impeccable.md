## Design Context

### Users
Vietnamese factory staff — warehouse operators, weighbridge attendants, and QC inspectors — at three manufacturing plants (NMXH, NMQM, NMCT). They are on 6–8 hour shifts, making time-pressured operational decisions: is the cargo queue backing up? Has this truck been weighed? Is this dossier complete? They use the interface daily, not occasionally. Cognitive load must be low, scan-speed must be high.

### Brand Personality
**Precise · Industrial · Trustworthy**

This is a tool, not a product. The emotional goal is calm confidence: the person at the weighbridge glances at the screen and immediately knows the operational state — no hunting, no ambiguity. The interface should feel like a well-calibrated instrument, not a consumer app.

References: Linear's information density, Vercel dashboard's calm neutral surfaces, Notion's data-view clarity.

### Aesthetic Direction
Minimal industrial workspace. Light mode only (dark mode supported as a secondary). Near-white page surface (#FAFAFA), pure-white cards, hairline borders, zero shadows. One accent color — deep blue (#1E40AF) — used sparingly for primary actions and active state. Status colors (green/amber/red) used only to communicate state, never as decoration. Monospace font (JetBrains Mono) for all data-like strings: plate numbers, IDs, timestamps. Inter for all prose. No gradients, no glass, no rounded-corner radii above 12px, no decorative illustrations.

Anti-references: consumer SaaS dashboards with gradient hero numbers, glassmorphism, large icons above every heading, all-caps section titles, AI-slop cyan-on-dark palettes.

### Design Principles
1. **Data is the foreground.** Chrome (borders, headers, sidebars) is hairline. Nothing decorative competes with the operational data.
2. **One accent per screen.** The primary button is the only filled-accent element. All other interactive elements use ghost or outlined styles.
3. **Monospace marks machine data.** Plate numbers, IDs, slip codes, timestamps in dense views — all JetBrains Mono. This lets staff read them at a glance without parsing context.
4. **Status communicates, never decorates.** Green/amber/red appear only on status badges, dot indicators, and 2px left-border callouts. Never as section fills or background gradients.
5. **Vietnamese first.** All labels, states, empty messages, and copy are in Vietnamese with full diacritics. The active plant is always visible in the header — it is the operational context for every piece of data.
